Object:AddPlayerExecutionRule(self, vRule, fManualCheck)
Unimplemented

This method or function is UNIMPLEMENTED!

This means it does not yet have functionality, but exists in code!

View source »

Adds an Execution Rule for a Player.

Parameters

  • self SafeNet.Object

    The SafeNet.Object you are modifying. This parameter is auto filled if you are using this as a method.

  • vRule

    What variable to pull, this can be a function, table, or string. A string will look for that field on the player's object, if it exists, it will pass. A table with string keys will do the same, but evaluate the value for the fields's entry. A function will return the values from the function to fManualCheck.

  • fManualCheck function

    fManualCheck will always run, if it exists. Returning an SafeNet.ERRLEV will run as if that SafeNet.ERRLEV has been reached.

    Passed parameters: (self[SafeNet.Object], client[Player], varargs[values retrieved]) fManualCheck will be passed anything retrieved by the function. Keep in mind, that the intended net message has not yet been sent to the server, as a result; network parameters are not yet available.

Returns

  • bool

    Whether or not the Execution Rule was successfully added or not.

Example Usage

safenetobj:AddPlayerExecutionRule("cool")
OR
safenetobj:AddPlayerExecutionRule({
   ['foo'] = false
   ['bar'] = true,
   ['bummer'] = 30
)
OR
safenetobj:AddPlayerExecutionRule(nil, function(self, client, mario, luigi)
     if mario then return false end
     if luigi then return true end
     return SafeNet.ERRLEV.LOCK
end)
