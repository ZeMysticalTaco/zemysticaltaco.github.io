Werwolf Gaming Documentation

Welcome to the documentation for Werwolf Gaming's development team.

Overview

This documentation serves as a hub for all of the information relating to the code of the servers.

Onboarding

Getting started is pretty simple, it's recommended you read the `Coding Standards` page before starting.

Clone your Repository

If you know how git works, and don't need this explained to you: You can skip this explanation and go down to Navigating your Codebase.

Create a GitHub account

https://github.com/

Downloading Github Desktop

First, if you don't have it already, download git. Click Here!

for the sake of brevity, you're being linked to GitHub Desktop, this minimizes the time taken to explain this to you. Follow the installation steps and login.

After Signing In

Now that you're signed in, we're going to clone the repository.

Picture

Click the top left drop down box.

Picture

Click Add and then clone repository.

Clone the Repository

Picture

If you did everything right, this should be a similar picture. Clone your repository to the location that's listed in your Repository's README.md For example, for HLRP, you would clone it to: SERVER/garrysmod/gamemodes/ixhl2p

Validating

To validate whether or not your cloning was successful, the root folder that you cloned the repository to should look identical to the branch you cloned it from.

Once you've validated everything is correct -- you're ready to move on.

Navigating your Codebase

As of right now, the documentation system is limited to HLRP. explain docs

Contributing Code to your Codebase

Before you Contribute

If you have been assigned a task to do already, and are going to contribute code to the Codebase:
Make sure you have read your Repository's Code Standards, and are following them properly. Do not push to `main` unless you have permission to. Failure to follow that rule may result in restrictions.

Building your Development Server [D/O]

In order to build your respective server's development environment, you can either trigger the job using your branch as an argument, or commit to the dev repository.
Using the `"DUMP_DATA"` argument will dump the development server's database and data/ folders, wiping the server effectively clean.
