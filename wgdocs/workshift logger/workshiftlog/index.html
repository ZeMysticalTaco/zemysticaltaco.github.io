ix.WWG.shift.SendLog
Network String

This is a network string, meaning it only exists to document existing net code.
The realm denotes which realm is intended to receive this message.

 View source »

Sends a WorkshiftLog to the client. This runs the necessary functions to reform the WorkshiftLog object on the client already, if you want to access it, it is stored in ix.shift.stored.

By default this will only send the client the Name and Character ID of the creator, any additional data must be sent yourself.

Network Data

  • arg uniqueID type number numberType UInt bits 32 readOrder 1

    The ID of the log being sent. This is an UNSIGNED INTEGER! Feeding this message negative numbers will destroy data integrity!

  • arg title type string readOrder 2

    The title of the log being sent.

  • arg body type string readOrder 3

    The body of the log being sent.

  • arg creatorName type string readOrder 4

    The name/source of the creator being sent.

  • arg creatorCharacterID type number numberType Int bits 32 readOrder 5

    The CharacterID of the creator. This will be -1 if there is no character ID!

  • arg time type number readOrder 6

GetVar(key, fallback)
Unimplemented Code

This is UNIMPLEMENTED! This means it is documented, but has not yet been written in code!

Function

This is a Function or a Method.
Internal

This is an internal function! You are able to use it, but you risk unintended side effects if used incorrectly.

View source »

Gets a Var on this WorkshiftLog This is used internally to reference variables, and returns raw variables, not sanitized ones. Since the object in memory is considered to be the "master object", this will return cached values after the first. If you need to actually get a stored database value without loading the WorkshiftLog into memory without an already existing method, you're probably using this PLUGIN wrong.

Parameters

  • arg key type string

    The key to the var.

  • arg fallback

    The fallback variable if this var is not valid.

Returns

  • any

    Whatever value is in the key's field.

    • OR
  • any

    Whatever is in the fallback field.

Initialize(categoryID, title, text, time, creator)
Unimplemented Code

Function

This is a Function or a Method.
Internal

This is an internal function! You are able to use it, but you risk unintended side effects if used incorrectly.

View source »

The base constructor class. You should use ix.shift.NewWorkshiftLog instead, this is the constructor class. Due to the asynchronous nature of databases:

If you want to run code each time one is created, you should use WorkshiftLogCreated.

This object's UniqueID may not immediately exist.

Parameters

  • arg categoryID type string

    The Category ID to insert this object into.

  • arg title type string

    The title of the workshift log.

  • arg text type string

    The text of the workshift log, can contain Multi-Lines.

  • arg time type number default: os.time()

    The time the log was created.

  • arg creator type Character default: -1

    Who created the log, if anyone.
